## Project overview

Memory Bank is a TypeScript/Node.js Claude Code plugin and package for semantic conversation memory. It provides:

- CLI entry points under `cli/` and `src/*-cli.ts`.
- An MCP server in `src/mcp-server.ts` exposing `search, read, search_facts, search_ontology, ask_avatar, trace_fact, graph_stats, cross_project_insights, explore_graph`.
- SQLite persistence through `better-sqlite3` plus `sqlite-vec` vector tables.
- Local embeddings from `@xenova/transformers` (`all-MiniLM-L6-v2`, 384 dimensions).
- Fact extraction/consolidation via `src/fact-extractor.ts`, `src/consolidator.ts`, `src/llm.ts`.
- Ontology graph storage/query through `src/ontology-db.ts` and `src/ontology-classifier.ts`.
- A readonly web dashboard in `ui/server.cjs` plus graph HTML assets in `docs/`.

## Coding conventions

- Root package is ESM (`"type": "module"`): TypeScript imports must include `.js` extensions for local modules.
- Do not use `require()` in `src/**/*.ts`; CommonJS is confined to `.cjs` runtime files such as `ui/server.cjs`.
- Prefer `import type` for type-only imports.
- Shared domain types belong in `src/types.ts`; DB row interfaces stay local to modules that query them.
- For `better-sqlite3`, cast rows to precise interfaces; do not use broad `as any` when a row shape is known.
- Always close `initDatabase()` connections with `finally` unless ownership is explicitly transferred.
- Keep `db.pragma('busy_timeout = 5000')` for runtime DB connections.
- Use parameterized SQL. LIKE queries must escape `%`, `_`, and `\` and include `ESCAPE '\'`.
- `sqlite-vec` virtual tables do not use upsert/replace; update with DELETE then INSERT in a transaction.
- File sync writes should stay atomic: temp file then rename.
- JSONL parsing should keep per-line `try/catch` tolerance for malformed records.
- Tests must isolate data using env overrides and temporary directories.
