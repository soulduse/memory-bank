Architecture Decisions

서버화가 아니라 운영체제화

Tenant isolation first

테넌트는 고객/회사 단위의 하드 격리 경계다. 모든 API, DB row, object key, vector payload, cache key는 tenant_id를 첫 번째 필터로 가져야 한다.

  • pool/silo/hybrid isolation mode
  • RBAC + ABAC + audit trail
  • cross-tenant test blocks deploy

Archive-first ingestion

검색·벡터·팩트 추출은 늦어질 수 있지만, 승인된 원본 이벤트 손실은 허용하지 않는다. 원본 아카이브와 큐 publish가 sidecar spool 삭제의 기준이다.

  • raw JSONL object archive
  • idempotency by terminal/session/event_seq
  • backpressure defers expensive work

Evidence-gated Loopy-Era

자가개선은 자동 생성이 아니라 검증된 승격이다. QA 실패·fix commit·정책 위반 시그널은 canary bundle, replay, load smoke, 승인을 통과해야 publish된다.

  • signed evidence contract
  • keep/discard/rollback lifecycle
  • no BLOCKED marker completion claim
Scope Model

회사→조직→팀→프로젝트→사용자→터미널까지 메모리 경계를 명시한다

Promotion rules

  • personal: user-only; never auto-promote without explicit policy event.
  • project: project members; maintainer approval or repeated evidence.
  • team: DLP/secret scan plus team admin approval.
  • org/company: steward/security approval, canary validation, rollback plan.
  • global_template: vendor-owned generic patterns only; never tenant data.

Hard Verification Gates

엔터프라이즈 준비 완료는 JSON 증거 파일이 PASS일 때만 말할 수 있다

Release invariant: load, tenant isolation, failover, security scan, policy rollback evidence가 모두 PASS가 아니면 클라우드 엔터프라이즈 readiness를 주장하지 않는다. Smoke test나 수동 확인은 HARD gate를 대체하지 못한다.

Migration Path

현재 로컬-first Memory Bank에서 클라우드 셀까지의 이행 순서

Phase 0

Contracts

event schema, tenant IDs, evidence JSON, policy contract를 먼저 고정한다.

Phase 1

Sidecar

local hook/sync를 offline spool + idempotent upload로 감싼다.

Phase 2

Ingest

auth, raw archive, event bus, Postgres metadata write path를 만든다.

Phase 3

Search

text/vector index workers와 cached context API를 분리한다.

Phase 4

Governance

org/team admin, scope promotion, DLP, audit, rollback을 붙인다.

Phase 5

Evidence

10k terminals, 15k QPS, tenant isolation, failover gates를 release blocker로 만든다.
