01 — Problem Reframe

분산 agent 통신을 늘리는 게 아니라, 공유 context plane으로 줄인다

enterprise-harness-complexity.html의 핵심 경고는 조직 확장에서 scope, governance, observability, multi-team coordination이 곱셈으로 복잡해진다는 것이다. 그래서 Memory Bank Cloud는 모든 Worker가 직접 합의·전파하는 구조 대신, Supabase MCP host가 scope별 context를 관리하고 Claude/Codex가 로그인 시 자동으로 읽는 구조로 단순화한다.

03 — Scope Model

company → org → team → project → personal

SCOPE 01
Company / Org
테넌트 hard boundary와 조직 운영 규칙. 모든 row는 tenant_id를 갖고 RLS 대상이 된다.

SCOPE 02
Team / Project

teamprojectcontext

Personal

04 — Supabase Host

MCP host 역할은 Supabase/Postgres + RLS + audit로 구성한다

IMPLEMENTED
Local contract slice
src/memory-bank-cloud.ts와 src/memory-bank-cloud-mcp.ts가 token/login/context bundle core를 testable하게 구현한다.
SUPABASE TARGET
Schema + RLS skeleton
docs/memory-bank-cloud-supabase-schema.sql가 tenants, memberships, context_entries, audit_events와 RLS policy의 시작점을 정의한다.
05 — MCP Tools

Cloud memory를 private MCP session contract로 고정한다

TOOL 01
memory_bank_cloud_issue_token
admin/control-plane 전용. issuer envelope를 검증한 뒤 issuer scope를 넘지 않는 login token만 발급한다.
TOOL 02
memory_bank_cloud_login
발급 토큰으로 로그인하고 Claude/Codex session에 issuer boundary로 clip된 account context와 memberships를 붙인다.
TOOL 03
memory_bank_cloud_put_context
personal/project/team/org/company scope에 context를 저장한다. scope 밖 write는 거부한다.
TOOL 04
memory_bank_cloud_get_context
현재 session token issuer boundary 안에서 볼 수 있는 context bundle만 자동 반환한다.
